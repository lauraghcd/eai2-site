EAI² 已建立扎实的视觉骨架（18 篇文章、5 知识区、新近重做的视觉化卡片设计），但**作为 SEO/AI Citation 入口的核心要素严重不足**。最大风险是「内容看着多、实际权重轻」：平均 883 字/篇远低于 2026 年 AI 搜索引擎倾向的 1500-2500 字标准；10/18 篇文章无内链导致知识图谱碎片化；零作者署名意味着 Google EEAT 信号缺失，AI 引擎会**直接降权**；缺少 Article schema markup 让站点对 AI 引擎几乎不可见。本次审核识别 **4 个 P0、4 个 P1、4 个 P2** 共 12 个待修事项，最关键的 4 项（schema、作者、TL;DR、内链）48 小时内可解决，能立刻提升 AdSense 审核通过率与未来流量天花板。

### Frontend Defects

**P0 (block ranking/citation):**

- **F-P0-1**: 站点根 `/` 无 meta description — Hugo 未生成（`layouts/index.html` 缺 SEO meta）。loss: 主页搜索摘要缺失
- **F-P0-2**: 文章页无 author 署名 — `single.html` line 17-22 only shows date/reading time。EEAT 失败。
- **F-P0-3**: 无 Article schema (JSON-LD) — `extend_head.html` 仅有 AdSense 脚本。AI 引擎不识别为 article。
- **F-P0-4**: 文章顶部无 TL;DR/Key Takeaways — AI 引擎在 540 字就停止读取，最重要内容应该在前 3 段。当前文章前 540 字基本是介绍性语句。

**P1:**

- **F-P1-1**: Learning Path 页（`/learning-path/`）UI 极朴素，纯文本 markdown，无 visual progress indicator。本应是流量入口之一。
- **F-P1-2**: 分类 landing 页（`/categories/fundamentals/` 等）只有简单介绍 + 文章列表，无该类目下知识地图、推荐顺序、覆盖度可视化。
- **F-P1-3**: 文章末尾无"相关文章"自动推荐区。读者读完即跳出。
- **F-P1-4**: 无邮件订阅 CTA（newsletter signup）— 即使有流量也无法沉淀用户。

**P2:**

- **F-P2-1**: 暗色模式色彩对比度未做 A11Y 测试（WCAG AA 检查）
- **F-P2-2**: 文章超过 1000 字应有 sticky TOC（侧边栏跟随滚动），目前仅有顶部 TOC
- **F-P2-3**: 移动端 article hero 内 emoji 叠在标题上（虽然透明度低但仍轻微干扰）
- **F-P2-4**: 文章 footer share 按钮位置太低，无 social proof（无 share count）

### Content Defects

**C-P0**: Article body 平均 883 字/篇，AI 搜索引擎和 Google E-E-A-T 都偏好 1500-2500 字。当前内容深度等同于"介绍性博客"而非"权威资源"。

**C-P0**: LeRobot Tutorial 551 字（最短）— 但这是引流型 hands-on 教程，最该详尽。竞品 Hugging Face 官方 LeRobot docs 单页 3000+ 字 + 大量代码。

**C-P1**: 18 篇文章中 5 篇没有结构化表格 — 2026 数据：comparison pages with 3+ tables 多得 25.7% AI 引用。

**C-P1**: 站点无任何 FAQ 块 — 错过 FAQ schema → 无法进 People Also Ask 和 AI Overview。

### Backend Defects

无 backend（纯静态站点）。Cloudflare Pages 自动部署一切正常。

### Journey Maps & Breakpoints

#### Persona 1: Python 工程师 → embodied AI 入门

```
Google "how to start with embodied AI" 
   → 进入 /posts/getting-started-embodied-ai-2026/
   → 看到 Hero 类别色（紫色 Fundamentals 📚）→ ✅ 视觉吸引
   → 滚动看内容 → ❌ 940 字过短，深度不够
   → 想看下一步 → ❌ 文章末尾无推荐 → 跳出
```

**Breakpoints:**
- 文章末尾无 "Next: Tools & Frameworks" 引导（Learning Path 是独立页，文章页未链接）
- 没有 prerequisite check（"读完这篇你应该会..."）
- LeRobot 教程被引用但无内链跳转

#### Persona 2: 研究员 → 找 VLA 模型对比

```
Google "GR00T vs pi0 vs OpenVLA"
   → 进入 /posts/vla-models-compared-2026/
   → ✅ 找到对比表格
   → 文章 752 字（最短之一）→ ❌ 不够深
   → 读完 → ❌ 无 cite-as / DOI / 论文引用
```

**Breakpoints:**
- 学术读者期待论文引用（arXiv links）
- 无 author bio → 不知作者是否权威
- 无 last-updated 日期视觉强调（实际 frontmatter 有但 hero 上没有）

#### Persona 3: VC → humanoid 公司调研

```
Google "AgiBot Unitree humanoid robot funding 2026"
   → 进入 /posts/embodied-ai-funding-tracker-2026/ 或 /china-humanoid-robot-companies/
   → ✅ 数据点丰富
   → 但 631 字（funding tracker 最短）→ ❌ 不够覆盖
   → ❌ 无可下载 PDF/data export
```

**Breakpoints:**
- 投资人想分享给团队 → 无 share / save 强提示
- 无 newsletter subscribe → 无法沉淀
- 文章没有公司 logo / 图表（数据可视化）

#### Persona 4: 大学讲师 → 教学资源

```
进入站点首页
   → ✅ 看到 Learning Path，认为体系化
   → 进入 /learning-path/
   → ❌ UI 朴素，无下载选项，无 print-friendly 版本
   → ❌ 无 Creative Commons license 标识 → 不敢直接给学生用
```

**Breakpoints:**
- 无 license 信息 → 教学复用障碍
- 无引用格式（BibTeX / APA）

### Critical UX Blockers

1. **无作者身份** — 4 个 persona 都需要判断"作者是谁、可信吗"
2. **无下一步引导** — 读者读完文章无 Next CTA，跳出率会很高
3. **无订阅入口** — 即使读者满意，也无法沉淀

### UX Improvement Opportunities

- 文章末尾自动 "Continue reading" 推荐（同类目最近 2 篇 + 跨类目互补 1 篇）
- Learning Path 改为可视化 step indicator + 进度跟踪（localStorage 即可，无需后端）
- 文章 hero 加 "Last updated" + "Reading effort" + "Prerequisites"

| Pi0 / GR00T 官方文档 | 模型厂商 | 第一手 | 我们做交叉对比 |

### 2026 SEO/AI Citation 标准（关键发现）

1. **AI 引擎在 ~540 字处停止 grounding** → TL;DR 必须前置
2. **Comparison pages with 3+ tables** 多 25.7% citations
3. **TOC for 1000+ words** 是 "table stakes"
4. **Schema markup is #1 audit gap**（我们恰好缺）
5. **作者 bio with credentials 影响 EEAT** → AI 引擎深度交叉验证 LinkedIn
6. **DA / referring domains > 32K → 3.5x more ChatGPT citations**
7. **Meta descriptions 140-160 chars** 每字必争

### Trend Alignment Recommendations

- 加 TL;DR / Key Takeaways 块到每篇文章前面（解决 540 字 grounding）
- 加 Comparison schema + 多做对比表（25.7% citation 红利）
- 加 Author 页 + bio with credentials（EEAT 解锁）
- 加 Article schema markup（修补 #1 gap）
- 长期：增加 referring domains（同行交换链接）
